Namespace Arrays

Arrays are a list of data. ixfx provides a number of functions for working with arrays in an immutable manner. This means that the input array is not changed.

For arrays of numbers:

  • average, max, min, total: Calculate average/max/min/total
  • averageWeighted: Calculate average, but applies a weighting function, eg to favour items at beginning of array
  • minMaxAvg: Find smallest, largest and average
  • maxIndex, minIndex: Return index of largest/smallest value
  • dotProduct: Returns the dot-product between two arrays
  • weight: Applies a weighting function to all values based on their index

Randomisation

  • randomIndex: Return a random array index
  • randomElement: Return a random value
  • randomPluck: Remove a random element from an array, returning it and the new array
  • shuffle: Returns a randomly-sorted copy of arra

Finding/accessing

  • filterBetween: Same as Array.filter but only looks within a specified index range
  • sample: Returns a new array with a random sampling of input
  • areValuesIdentical: Returns true if all the values in the array are identical

Changing the shape

  • ensureLength: Returns a copy of array with designated length, either padding it out or truncating as necessary
  • groupBy: Groups data into a new Map
  • interleave: Flattens several arrays into one, interleaving their values.
  • remove: Remove an item by index
  • without: Returns an array with specified value omitted
  • zip: Groups together elements from several arrays based on their index

areValuesIdentical

  • areValuesIdentical<V>(array: readonly V[], equality?: IsEqual<V>): boolean

  • Returns true if all the contents of the array are identical.

    example

    Uses default equality function:

    import { areValuesIdentical } from 'https://unpkg.com/ixfx/dist/arrays.js';

    const a1 = [10, 10, 10];
    areValuesIdentical(a1); // True

    const a2 = [ {name:`Jane`}, {name:`John} ];
    areValuesIdentical(a2); // True, because JSON version captures value

    If we want to compare by value for objects that aren't readily converted to JSON, you need to provide a function:

    areValuesIdentical(someArray, (a, b) => {
     return (a.eventType === b.eventType);
    });

    Type parameters

    • V

    Parameters

    • array: readonly V[]

      Array

    • Optional equality: IsEqual<V>

      Equality checker. Uses string-conversion checking by default

    Returns boolean

average

  • average(data: readonly number[]): number

  • Calculates the average of all numbers in an array. Array items which aren't a valid number are ignored and do not factor into averaging.

averageWeighted

  • averageWeighted(data: readonly number[], weightings: readonly number[] | EasingFn): number

  • Computes an average of an array with a set of weights applied.

    Weights can be provided as an array, expected to be on 0..1 scale, with indexes matched up to input data. Ie. data at index 2 will be weighed by index 2 in the weightings array.

    import { Arrays } from 'https://unpkg.com/ixfx/dist/collections.js';
    // All items weighted evenly
    Arrays.averageWeighted([1,2,3], [1,1,1]); // 2

    // First item has full weight, second half, third quarter
    Arrays.averageWeighted([1,2,3], [1, 0.5, 0.25]); // 1.57

    // With reversed weighting of [0.25,0.5,1] value is 2.42

    A function can alternatively be provided to compute the weighting based on array index, via weight.

    Arrays.averageWeighted[1,2,3], Easings.gaussian()); // 2.0

    This is the same as:

    import { Arrays } from 'https://unpkg.com/ixfx/dist/collections.js';
    import { Easings } from 'https://unpkg.com/ixfx/dist/modulation.js';

    const data = [1,2,3];
    const w = Arrays.weight(data, Easings.gaussian());
    const avg = Arrays.averageWeighted(data, w); // 2.0

    Parameters

    • data: readonly number[]

      Data to average

    • weightings: readonly number[] | EasingFn

      Array of weightings that match up to data array, or an easing function

    Returns number

ensureLength

  • ensureLength<V>(data: readonly V[], length: number, expand?: "undefined" | "repeat" | "first" | "last"): readonly V[]

  • Returns an copy of data with specified length. If the input array is too long, it is truncated.

    If the input array is too short, it will be expanded based on the expand strategy:

    • 'undefined': fill with undefined
    • 'repeat': repeat array elements, starting from position 0
    • 'first': continually use first element
    • 'last': continually use last element
    import { ensureLength } from 'https://unpkg.com/ixfx/dist/arrays.js';

    ensureLength([1,2,3], 2); // [1,2]
    ensureLength([1,2,3], 5, `undefined`); // [1,2,3,undefined,undefined]
    ensureLength([1,2,3], 5, `repeat`);    // [1,2,3,1,2]
    ensureLength([1,2,3], 5, `first`);     // [1,2,3,1,1]
    ensureLength([1,2,3], 5, `last`);      // [1,2,3,3,3]

    Type parameters

    • V

      Type of array

    Parameters

    • data: readonly V[]

      Input array to expand

    • length: number

      Desired length

    • expand: "undefined" | "repeat" | "first" | "last" = ...

      Expand strategy

    Returns readonly V[]

groupBy

  • groupBy<K, V>(array: readonly V[], grouper: (item: V) => K): Map<K, V[]>

  • Groups data by a function grouper, returning data as a map with string keys and array values. Multiple values can be assigned to the same group.

    grouper must yield a string designated group for a given item.

    example
    import { groupBy } from 'https://unpkg.com/ixfx/dist/arrays.js';

    const data = [
     { age: 39, city: `London` }
     { age: 14, city: `Copenhagen` }
     { age: 23, city: `Stockholm` }
     { age: 56, city: `London` }
    ];

    // Whatever the function returns will be the designated group
    // for an item
    const map = groupBy(data, item => data.city);

    This yields a Map with keys London, Stockholm and Copenhagen, and the corresponding values.

    London: [{ age: 39, city: `London` }, { age: 56, city: `London` }]
    Stockhom: [{ age: 23, city: `Stockholm` }]
    Copenhagen: [{ age: 14, city: `Copenhagen` }]

    Type parameters

      Type of key to group by. Typically string.

    • V

      Type of values

    Parameters

      Array to group

    • grouper: (item: V) => K

      Function that returns a key for a given item

weight

  • weight(data: readonly number[], fn?: (relativePos: number) => number): readonly number[]

  • Applies a function fn to the elements of an array, weighting them based on their relative position.

    import { Arrays } from 'https://unpkg.com/ixfx/dist/collections.js';

    // Six items
    Arrays.weight([1,1,1,1,1,1], Easings.gaussian());

    // Yields:
    // [0.02, 0.244, 0.85, 0.85, 0.244, 0.02]

    fn is expected to map (0..1) => (0..1), such as an Modulation.Easings.EasingFn. The input to the fn is the relative position of an element. Thus the first element will be 0, the middle 0.5 and so on. The output of fn is then multiplied by the original value.

    In the below example (which is also the default if fn is not specified), the relative position is how values are weighted:

    Arrays.weight([1,1,1,1,1,1], (relativePos) => relativePos);
    // Yields:
    // [0, 0.2, 0.4, 0.6, 0.8, 1]

    Non-numbers in data will be silently ignored (this filtering happens first, so relative index values are sane still).

    Parameters

    • data: readonly number[]

      Array of numbers

    • Optional fn: (relativePos: number) => number

      Returns a weighting based on the given relative position. If unspecified, (x) => x is used.

        • (relativePos: number): number

        • Parameters

          • relativePos: number

          Returns number

    Returns readonly number[]

without

  • without<V>(data: readonly V[], value: V, comparer?: IsEqual<V>): readonly V[]

  • Returns an array with a value omitted. If value is not found, result will be a copy of input. Value checking is completed via the provided comparer function. By default checking whether a === b. To compare based on value, use the isEqualValueDefault comparer.

    example
    import { without } from 'https://unpkg.com/ixfx/dist/arrays.js';

    const data = [100, 20, 40];
    const filtered = without(data, 20); // [100, 40]
    example

    Using value-based comparison

    import { without } from 'https://unpkg.com/ixfx/dist/arrays.js';

    const data = [{name: `Alice`}, {name:`Sam`}];

    // This wouldn't work as expected, because the default comparer uses instance,
    // not value:
    without(data, {name: `Alice`});

    // So instead we can use a value comparer:
    without(data, {name:`Alice`}, isEqualValueDefault);
    example

    Use a function

    import { without } from 'https://unpkg.com/ixfx/dist/arrays.js';

    const data = [{name: `Alice`}, {name:`Sam`}];
    without(data, {name:`ALICE`}, (a, b) => {
     return (a.name.toLowerCase() === b.name.toLowerCase());
    });

    Consider remove to remove an item by index.

    Type parameters

    • V

      Type of array items

    Parameters

    • data: readonly V[]

      Source array

    • value: V

      Value to remove

    • comparer: IsEqual<V> = isEqualDefault

      Comparison function. If not provided Util.isEqualDefault is used, which compares using ===

    Returns readonly V[]

    Copy of array without value.

zip

  • zip(...arrays: readonly any[]): readonly any[]

  • Zip ombines the elements of two or more arrays based on their index.

    import { zip } from 'https://unpkg.com/ixfx/dist/arrays.js';

    const a = [1,2,3];
    const b = [`red`, `blue`, `green`];

    const c = zip(a, b);
    // Yields:
    // [
    //   [1, `red`],
    //   [2, `blue`],
    //   [3, `green`]
    // ]

    Typically the arrays you zip together are all about the same logical item. Eg, in the above example perhaps a is size and b is colour. So thing #1 (at array index 0) is a red thing of size 1. Before zipping we'd access it by a[0] and b[0]. After zipping, we'd have c[0], which is array of [1, red].

    Parameters

    • Rest ...arrays: readonly any[]

    Returns readonly any[]

    Zipped together array
