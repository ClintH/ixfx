EvalOpts

EvalOpts: { assumeExclusive?: boolean; timeoutMs?: number }

Options for code evaluation

Type declaration

  • Optional Readonly assumeExclusive?: boolean

    If true (default), it assumes that anything received from the board is a response to the eval

  • Optional Readonly timeoutMs?: number

    Milliseconds to wait before giving up on well-formed reply. 5 seconds is the default.

Options

Options: { debug?: boolean; evalTimeoutMs?: number; name?: string }

Options for device

Type declaration

  • Optional Readonly debug?: boolean

    If true, additional logging information is printed

  • Optional Readonly evalTimeoutMs?: number

    Default milliseconds to wait before giving up on a well-formed reply. 5 seconds is the default.

  • Optional Readonly name?: string

    Name of device. Only used for printing log mesages to the console

Functions

connect

  • connect(): Promise<EspruinoDevice>

  • An Espruino BLE-connection

    Use the puck function to initialise and connect to a Puck.js. It must be called in a UI event handler for browser security reasons.

    const e = await puck();

    Listen for events:

    // Received something
    e.addEventListener(`data`, d => console.log(d.data));
    // Monitor connection state
    e.addEventListener(`change`, c => console.log(`${d.priorState} -> ${d.newState}`));

    Write to the device (note the \n for a new line at the end of the string). This will execute the code on the Espruino.

    e.write(`digitalPulse(LED1,1,[10,500,10,500,10]);\n`);

    Run some code and return result:

    const result = await e.eval(`2+2\n`);

    Returns Promise<EspruinoDevice>

    Returns a connected instance, or throws exception if user cancelled or could not connect.
