Namespace Maps

Maps associate keys with values. Several helper functions are provided for working with the standard JS Map class.

ixfx also includes MapMutable, MapImmutable

Overview:

  • getOrGenerate: Solves a common scenario of wanting a value by a particular key, or generating it if it doesn't exist
  • filter: Yields values in map that match a predicate
  • find: Finds the first value that matches a predicate, or undefined if nothing found
  • hasAnyValue: Searches through all keys, returning true if any occurence of value was found

Transformations:

  • toArray: Returns the values of the map as an array
  • mapToArray: Applies a function to convert a map's values to an array
  • mapToObj: Coverts a Map to a plain object, useful for JSON serialising.
  • mapToObjTransform: Converts a map to a plain object, but applying a function to values
  • transformMap: Like Array.map, but for Maps. Useful for generating a map as a transform of an input map.
  • zipKeyValue: Given an array of keys and values, combines them together into a map

addUniqueByHash

  • addUniqueByHash<V>(set: undefined | ReadonlyMap<string, V>, hashFunc: ToString<V>, ...values: readonly V[]): Map<any, any>

  • Adds items to a map only if their key doesn't already exist

    Uses provided Util.ToString function to create keys for items. Item is only added if it doesn't already exist. Thus the older item wins out, versus normal Map.set where the newest wins.

    example
    const map = new Map();
    const peopleArray = [ _some people objects..._];
    addUniqueByHash(map, p => p.name, ...peopleArray);

getOrGenerate

  • getOrGenerate<K, V, Z>(map: Map<K, V>, fn: (key: K, args?: Z) => V | Promise<V>): GetOrGenerate<K, V, Z>

  • Returns a function that fetches a value from a map, or generates and sets it if not present. Undefined is never returned, because if fn yields that, an error is thrown.

    See getOrGenerateSync for a synchronous version.

    const m = getOrGenerate(new Map(), (key) => {
     return key.toUppercase();
    });

    // Not contained in map, so it will run the uppercase function,
    // setting the value to the key 'hello'.
    const v = await m(`hello`);  // Yields 'HELLO'
    const v1 = await m(`hello`); // Value exists, so it is returned ('HELLO')

hasAnyValue

  • hasAnyValue<K, V>(map: ReadonlyMap<K, V>, value: V, comparer: IsEqual<V>): boolean

  • Returns true if any key contains value, based on the provided comparer function. Use hasKeyValue if you only want to find a value under a certain key.

    Having a comparer function is useful to check by value rather than object reference.

    example

    Finds value where name is 'samantha', regardless of other properties

    hasAnyValue(map, {name:`samantha`}, (a, b) => a.name === b.name);

    Works by comparing value against all values contained in map for equality using the provided comparer.

    Type parameters

    • comparer: IsEqual<V>

    Returns boolean

    True if value is found

mapToArray

  • mapToArray<K, V, R>(m: ReadonlyMap<K, V>, transformer: (key: K, item: V) => R): readonly R[]

  • Converts Map to Array with a provided transformer function. Useful for plucking out certain properties from contained values and for creating a new map based on transformed values from an input map.

    example

    Get an array of ages from a map of Person objects

    let person = { age: 29, name: `John`};
    map.add(person.name, person);

    const ages = mapToArray(map, (key, person) => person.age);
    // [29, ...]

    In the above example, the transformer function returns a number, but it could just as well return a transformed version of the input:

    // Return with random heights and uppercased name
    mapToArray(map, (key, person) => ({
     ...person,
     height: Math.random(),
     name: person.name.toUpperCase();
    }))
    // Yields:
    // [{height: 0.12, age: 29, name: "JOHN"}, ...]

mapToObjTransform

  • mapToObjTransform<T, K>(m: ReadonlyMap<string, T>, valueTransform: (value: T) => K): {}

  • Converts a map to a simple object, transforming from type T to K as it does so. If no transforms are needed, use mapToObj.

    const map = new Map();
    map.set(`name`, `Alice`);
    map.set(`pet`, `dog`);

    const o = mapToObjTransform(map, v => {
     ...v,
     registered: true
    });

    // Yields: { name: `Alice`, pet: `dog`, registered: true }

    If the goal is to create a new map with transformed values, use transformMap.

transformMap

  • transformMap<K, V, R>(source: ReadonlyMap<K, V>, transformer: (value: V, key: K) => R): Map<K, R>

  • Like Array.map, but for a Map. Transforms from Map<K,V> to Map<K,R>, returning as a new Map.

    example
    const mapOfStrings = new Map();
    mapOfStrings.set(`a`, `10`);
    mapOfStrings.get(`a`); // Yields `10` (a string)

    // Convert a map of string->string to string->number
    const mapOfInts = transformMap(mapOfStrings, (value, key) => parseInt(value));

    mapOfInts.get(`a`); // Yields 10 (a proper number)

    If you want to combine values into a single object, consider instead mapToObjTransform.

    Type parameters

    • K

      Type of keys (generally a string)

    • V

      Type of input map values

    • R

      Type of output map values

    Parameters

    • transformer: (value: V, key: K) => R
        • (value: V, key: K): R

        • Parameters

          • value: V
          • key: K

          Returns R

    Returns Map<K, R>
