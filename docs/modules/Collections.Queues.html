Namespace Queues

Queues store items in the order in which they are added.

Stacks and queues can be helpful when it's necessary to process data in order, but each one has slightly different behaviour.

Like lining up at a bakery, the oldest items (at the front of the queue) are removed before the newest items (at the end of the queue). This is different to Stacks, where the newest item (on top) is removed before the oldest items (at the bottom).

The ixfx implementations allow you to set a capacity limit with three policies for how items are evicted.

Create queues with queue or queueMutable. These return a Queue or QueueMutable respectively.

QueueOpts

QueueOpts: { capacity?: number; discardPolicy?: DiscardPolicy }

Queue options.

example

Cap size to 5 items, throwing away newest items already in queue.

const q = queue({capacity: 5, discardPolicy: `newer`});

Type declaration

  • Optional Readonly capacity?: number

    Capcity limit

  • Optional Readonly discardPolicy?: DiscardPolicy

    Default is additions, meaning new items are discarded.

    older: Removes items front of the queue (ie older items are discarded)

    newer: Remove from rear of queue to make space for new items (ie newer items are discarded)

    additions: Only adds new items that there are room for (ie. brand new items are discarded)

Functions

Const queue

  • queue<V>(opts?: QueueOpts, ...startingItems: readonly V[]): Queue<V>

  • Returns an immutable queue. Queues are useful if you want to treat 'older' or 'newer' items differently. Enqueing adds items at the back of the queue, while dequeing removes items from the front (ie. the oldest).

    let q = queue();           // Create
    q = q.enqueue(`a`, `b`);   // Add two strings
    const front = q.peek();    // `a` is at the front of queue (oldest)
    q = q.dequeue();           // q now just consists of `b`
    example

    Cap size to 5 items, throwing away newest items already in queue.

    const q = queue({capacity: 5, discardPolicy: `newer`});

    Type parameters

    • V

      Data type of items

    Parameters

    • opts: QueueOpts = {}
    • Rest ...startingItems: readonly V[]

      Index 0 is the front of the queue

    Returns Queue<V>

    A new queue

Const queueMutable

  • queueMutable<V>(opts?: QueueOpts, ...startingItems: readonly V[]): QueueMutable<V>

  • Returns a mutable queue. Queues are useful if you want to treat 'older' or 'newer' items differently. Enqueing adds items at the back of the queue, while dequeing removes items from the front (ie. the oldest).

    const q = queue();       // Create
    q.enqueue(`a`, `b`);     // Add two strings
    const front = q.dequeue();  // `a` is at the front of queue (oldest)
    example

    Cap size to 5 items, throwing away newest items already in queue.

    const q = queue({capacity: 5, discardPolicy: `newer`});

    Type parameters

    • V

      Data type of items

    Parameters

    • opts: QueueOpts = {}
    • Rest ...startingItems: readonly V[]

      Items are added in array order. So first item will be at the front of the queue.

    Returns QueueMutable<V>
